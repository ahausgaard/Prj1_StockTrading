Adapter Pattern

Få inkompatible systemer til at tale sammen

Software Design og Test

Hvilket problem løser dette?

  • Inkompatible interfaces: Du har to systemer, der burde kunne arbejde sammen, men deres metoder og datatyper passer ikke overens.
  • Tredjeparts-biblioteker: Du vil bruge et eksternt API, men du har ikke adgang til kildekoden, så du kan ikke bare rette det til at passe ind i dit system.
  • Legacy Kode: Du opgraderer et system, men er nødt til at understøtte en gammel kernekomponent, som ikke må ændres.

Filosofien bag

"Sæt et mellemstik i stikkontakten."

Wrapper-tankegangen.

Tænk på en rejseadapter: Dit danske stik passer ikke i den amerikanske stikkontakt. Du bygger ikke en ny bærbar, og du trækker ikke nye ledninger i væggen. Du sætter blot en adapter imellem, der oversætter formattet.

Løsningen i praksis (Designet)

Vi opdeler systemet i tre roller:

  • Target (Interfacet): Sådan som *vores* system forventer at snakke med komponenten.
  • Adaptee (Den Inkompatible): Den tredjeparts-klasse, vi gerne vil bruge, men som taler "det forkerte sprog".
  • Adapter: Klassen i midten. Den implementerer *Target* og kalder i smug på *Adaptee*.

Eksempel: Med Design (Det rene snit)

// 1. Vores Target Interface (Vores systems sprog)
public interface IBetaling {
    void GennemførBetaling(decimal beløb);
}

// 2. Adaptee: 3. parts API (Kan ikke rettes!)
public class StripeExternalAPI {
    public void Charge(float amount, string currency) { /* ... */ }
}

// 3. Adapteren (Oversætteren)
public class StripeAdapter : IBetaling {
    private StripeExternalAPI _stripe = new StripeExternalAPI();

    public void GennemførBetaling(decimal beløb) {
        // Oversætter fra vores system til eksternt format
        _stripe.Charge((float)beløb, "DKK");
    }
}

Hvordan ser det ud UDEN designet?

Uden adapteren bløder tredjepartskoden direkte ind i vores forretningslogik.

public class OrdreBehandler {
    // TÆT KOBLING: Vores forretningslogik er låst til Stripe!
    private StripeExternalAPI _api = new StripeExternalAPI();

    public void AfslutOrdre(Ordre ordre) {
        // Kernesystemet tvinges til at kende eksterne datatyper
        float stripeAmount = (float)ordre.TotalPris;
        string valuta = "DKK";

        _api.Charge(stripeAmount, valuta);
    }
}

Konsekvensen

Vendor Lock-in og Skrøbelighed.
