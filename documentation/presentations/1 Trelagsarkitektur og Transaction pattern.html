Filosofien bag Transaction Script

Én brugerhandling = Ét script (én metode).

Implementering:

Presentation: src/presentation/controllers/BuyStockController.java

Løsningen i kode: Trelag + Script

Se: src/presentation/controllers/BuyStockController.java

// 1. UI Laget (JavaFX Controller - modtager brugerhandling)
@FXML private void onBuyButtonClicked() {
    // Delegerer til viewModel (ikke direkte til service!)
    viewModel.buyShares();
}

// 2. Logik Laget (Business Service - Transaction Script)
public void buyShares(BuySharesRequest request) {
    uow.begin();  // Start transaktion
    try {
        Stock stock = stockDAO.getBySymbol(request.stockSymbol());
        if (stock == null) throw new IllegalArgumentException(...);

        Portfolio portfolio = portfolioDAO.getById(request.portfolioId());
        BigDecimal totalAmount = stock.getCurrentPrice()
            .multiply(new BigDecimal(request.quantity()));
        BigDecimal fee = feeStrategy.calculateFee(totalAmount);

        portfolio.setCurrentBalance(
            portfolio.getCurrentBalance().subtract(totalAmount.add(fee)));

        portfolioDAO.update(portfolio);
        uow.commit();  // Sikrer atomaritet
    } catch (Exception e) {
        uow.rollback();
    }
}

Hvordan ser det ud UDEN designet?

Anti-mønsteret: Smart UI / Pærevælling — Alt i knap-klikket!

// FORKERT: Alt blandet sammen direkte i Controller
@FXML private void onBuyButtonClicked() {
    // Direkte fil I/O i UI-handler!
    String json = Files.readString(Path.of("stocks.json"));
    Stock stock = parseStock(json);

    if (stock == null) {
        alertLabel.setText("Stock not found!");
        return;
    }

    int quantity = Integer.parseInt(quantityField.getText());
    BigDecimal cost = stock.getPrice().multiply(quantity);

    // Direkte fil-opdatering uden transaktion!
    Portfolio portfolio = loadPortfolioFromFile();
    portfolio.setBalance(portfolio.getBalance().subtract(cost));
    savePortfolioToFile(portfolio);  // ← Hvis denne fejler, er penge væk!

    // Direkte UI manipulation
    balanceLabel.setText(portfolio.getBalance().toString());
}
// Umuligt at teste uden at gemme/læse rigtige filer!

Konsekvensen af "Smart UI"

  • Umuligt at genbruge: Logik i UI-handler
  • Sikkerhedsrisiko: Fil I/O blandet i UI
  • Test-mareridt: Kan ikke unit-teste uden rigtige filer
  • Atomaritet tabt: Ingen rollback mulighed

Testbarhed i Trelagsarkitektur

Ved at adskille lagene kan vi isolere logikken og teste den uden infrastruktur.

I testen af BuySharesService.buyShares(), bytter vi de rigtige DAOs ud med Mock DAOs.

// Unit test - Ingen fil-I/O, ingen database!
@Test void buyShares_validPurchase_updatesBuyersBalance() {
    // Arrange: Mock DAOs - INGEN database!
    MockStockDAO stockDAO = new MockStockDAO();
    MockPortfolioDAO portfolioDAO = new MockPortfolioDAO();
    MockUnitOfWork uow = new MockUnitOfWork();

    BuySharesService service = new BuySharesService(
        stockDAO, portfolioDAO, ..., uow);

    // Act
    service.buyShares(new BuySharesRequest(...));

    // Assert - Tests uden infrastruktur, ~50ms
    assertTrue(portfolioDAO.updateWasCalled());
    assertEquals(1, uow.getCommitCount());
}

Opsummering

  • Trelagsarkitektur: Deler koden i UI, Logik og Data. Giver overblik og genbrugelighed.
  • Transaction Script: Simpel, procedural organisering af logik. Én handling = ét script.
  • Testbarhed: Lagdelingen tillader os at mocke databasen og teste forretningsreglerne isoleret.
