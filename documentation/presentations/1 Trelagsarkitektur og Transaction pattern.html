Transaction Script

Én brugerhandling = ét script

Implementering: Transaction Script

Se: src/business/services/trading/BuySharesService.java

Alternativet: Domain Model

I Domain Model bor logikken inde i domæneobjekterne — ikke i en service udefra.

// Transaction Script: service orkestrerer alt udefra
buySharesService.buyShares(request);
// → henter Portfolio, Stock, beregner gebyr, skriver 3 DAOs

// Domain Model: objektet ved selv hvad "køb" betyder
portfolio.buy(stock, quantity, feeStrategy);
// → Portfolio validerer, beregner og opdaterer sig selv
  • Transaction Script: Dumme objekter — smart service
  • Domain Model: Smarte objekter — tynd service

Opsummering

  • Trelagsarkitektur: Deler koden i Præsentation, Logik og Data — ét ansvar per lag
  • Transaction Script: Én handling = ét script — proceduralt, lineært, nemt at forstå
  • Testbarhed: Lagdelingen gør det muligt at mocke DAOs og teste forretningslogik isoleret
