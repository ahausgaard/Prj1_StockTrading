Software Design og Test

Unit Test vs. Integration Test

  • Unit Test: Tester én isoleret metode med mocks
  • Integration Test: Tester moduler sammen (f.eks. service + rigtig database)

Unit Test Strategi: Boundary Value Analysis

Gien eksempler fra: test/unit/business/services/trading/

// Boundary Value Analysis - test grænserne
@Test void percentageMinimumFeeStrategy_usesMinimumWhenLowerThanPercentage() {
    FeeStrategy strategy = new PercentageMinimumFeeStrategy(
        0.01,           // 1% gebyr
        BigDecimal.valueOf(15)  // 15 DKK minimum
    );

    // Test grænsværdier:
    // Under minimum (5 DKK < 15 DKK) → bør returnere 15
    BigDecimal fee = strategy.calculateFee(new BigDecimal("500"));
    assertEquals(new BigDecimal("15"), fee);
}

// Over minimum (50 DKK > 15 DKK) → bør returnere 50
BigDecimal fee2 = strategy.calculateFee(new BigDecimal("5000"));
assertEquals(new BigDecimal("50"), fee2);

Integration Test Strategi: Bottom-Up

  • Start forneden: Test Database/Repository-laget først
  • Byg op: Test Service-lagog Business Logic oven på
  • Fordel: Fanger DB-fejl tidligt, før de når UI'et

Opsummering

  • ✅ V-Modellen: Kravspec/Design på venstre, Test på højre
  • ✅ Unit Tests: Isoleret logik med Boundary Value Analysis
  • ✅ Integration Tests: Bottom-up fra DAOs til Services
  • ✅ Ingen overlap: Unit tester logik, Integration tester integration
